<Aside title="Adopting Den for NixOS and nix-Darwin flakes" icon="nix">
Den adapts to your Nix choices. This guide shows how to **integrate** Den without disrupting your **existing flake**, allowing an smooth, incremental adotpion path.

If you want a **Den only** flake example, take a look at `templates/minimal` or if you are using flake-parts see `templates/default`.

This is intended for people using a `flake.nix` file
that exposes `nixosConfigurations`, `darwinConfigurations` or `homeConfigurations`.

This guide assumes you have familiarity with:
- [Nix Flakes](https://nix.dev/concepts/flakes)
- The `flakes nix-command` experimental features enabled.
</Aside>

Suppose you have a flake like this with a `nixosConfigurations.igloo`:

```nix
# flake.nix
{
  inputs.nixpkgs = "github:nixos/nixpkgs/nixpkgs-unstable";
  inputs.darwin = "github:nix-darwin/nix-darwin";

  outputs = inputs: 
  {
    # all your config, most importantly a NixOS like `igloo`:
    nixosConfigurations.igloo = lib.nixosSystem { 
      modules = [ 
         <all your curent NixOS modules> 
      ];
    };
    darwinConfigurations.iceberg = darwinSystem {
      modules = [
         <all your current nix-Darwin modules>
      ];
    };
  };
}
```

### Step 1. A basic `modules/den.nix`

<Aside>
See [*From Zero To Den*](/guides/from-zero-to-den) for all the defails about this.

Also read the **No Monoliths** aside in that page about splitting this example configuration.
</Aside>

```nix
# modules/den.nix
{ inputs, den, lib, ... }: {
  imports = [ inputs.den.flakeModule ];

  den.schema.user.classes = lib.mkDefault [ "homeManager" ];

  den.hosts.x86_64-linux.igloo.users.tux = {};

  den.hosts.aarch64-darwin.iceberg.users.tux = {};

  den.aspects.igloo = {
    includes = [ den.provides.hostname ];
    nixos = { pkgs, ... }: { environment.systemPackages = [ pkgs.hello ]; };
  };

  den.aspects.tux = {
    includes = [ den.provides.define-user den.provides.primary-user ];
    homeManager = { pkgs, ... }: { packages = [ pkgs.vim ]; };
  };
}
```

Den batteries like `den.provides.define-user` and `den.provides.primary-user` work cross-platform, on NixOS, Darwin and WSL.

### Step 2. Use `igloo` NixOS module in your **current flake**

```diff lang="nix" ".mainModule"
# flake.nix
{
  inputs.nixpkgs = "github:nixos/nixpkgs/nixpkgs-unstable";
  inputs.darwin = "github:nix-darwin/nix-darwin";
+  inputs.import-tree.url = "github:vic/import-tree";
+  inputs.den.url = "github:vic/den";
  
  outputs = inputs: 
  let
+   den = (inputs.nixpkgs.lib.evalModules {
+     modules = [ (inputs.import-tree ./modules) ];
+     specialArgs.inputs = inputs;
+   }).config;

+   inherit (den.den.hosts.x86_64-linux) igloo;
+   inherit (den.den.hosts.aarch64-darwin) iceberg;
  in
  {
    # all your config, most importantly a NixOS like `igloo`:
    nixosConfigurations.igloo = nixosSystem { 
      modules = [ 
         <all your curent NixOS modules> 
+         igloo.mainModule
      ];
    };
    darwinConfigurations.iceberg = darwinSystem {
      modules = [
         <all your current nix-Darwin modules>
+         iceberg.mainModule
      ];
    };
  };
}
```

## Migrating with flake-parts.

If you are using `flake-parts` on your current configuration,
you will also need the following at the previous step:

```diff lang=nix
    den = (inputs.nixpkgs.lib.evalModules {
      modules = [ 
        (inputs.import-tree ./modules) 
+       inputs.den.flakeOutputs.flake
      ];
      specialArgs.inputs = inputs;
    }).config;
```

The `flakeOutputs.flake` module defines a flake-parts
compatible top-level `flake` output to be used by Den.

> See [output reference](/reference/output) for more on this.

## Once Migration Completed

Once you have moved all your host configurations into `modules/`,
and no longer have a need for mixin modules into nixosConfigurations,
you can have a flake like this:

```nix "den.flake.nixosConfigurations"
# flake.nix
{
  inputs.nixpkgs = "github:nixos/nixpkgs/nixpkgs-unstable";
  inputs.darwin = "github:nix-darwin/nix-darwin";
  inputs.import-tree.url = "github:vic/import-tree";
  inputs.den.url = "github:vic/den";
  
  outputs = inputs: 
  let
   den = (inputs.nixpkgs.lib.evalModules {
     modules = [ (inputs.import-tree ./modules) ];
     specialArgs.inputs = inputs;
   }).config;
  in
  {
    inherit (den.flake) nixosConfigurations darwinConfigurations;

    # all other flake outputs and stuff
  };
}
```

## Den-only flake outputs

Even more, if you have defined all flake outputs as Dendritic
`flake` outputs inside `modules/` your final `flake.nix` becomes:

```nix "config.flake"
# flake.nix
{
  inputs.nixpkgs = "github:nixos/nixpkgs/nixpkgs-unstable";
  inputs.darwin = "github:nix-darwin/nix-darwin";
  inputs.import-tree.url = "github:vic/import-tree";
  inputs.den.url = "github:vic/den";
  
  outputs = inputs: 
   (inputs.nixpkgs.lib.evalModules {
     modules = [ (inputs.import-tree ./modules) ];
     specialArgs.inputs = inputs;
   }).config.flake;
}
```
