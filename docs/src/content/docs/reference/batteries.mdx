Den ships reusable aspect providers under `den.provides` (aliased `den._`).
Each battery contributes NixOS/Darwin/home-manager modules to hosts and
users that include them via `den.ctx`.

## System batteries

### `den._.define-user`

Creates OS-level user accounts (`users.users.<name>`) with `isNormalUser`,
`home`, and group. Sets `home-manager.users.<name>` when home-manager
integration is active.

### `den._.os-user`

Assigns a user into the host by setting group membership and ensuring
the user account exists at the OS level.

### `den._.primary-user`

Marks a user as the primary user of a host. Sets `nix.settings.trusted-users`
and can configure auto-login or default shell based on other batteries.

### `den._.user-shell`

Sets the user's login shell. Reads `user.shell` and configures both
`users.users.<name>.shell` and the home-manager `programs.<shell>.enable`.

### `den._.tty-autologin`

Configures TTY auto-login for the primary user via
`services.getty.autologinUser`.

### `den._.wsl`

WSL-specific configuration. Enables `wsl.enable`, sets the default user,
and adjusts networking for WSL2 environments.

### `den._.forward`

Forwards aspect configuration from one aspect to another. Used to chain
aspects without duplicating declarations.

### `den._.import-tree`

Uses `inputs.import-tree` to recursively import a directory tree as
aspect modules. Enables file-system-driven aspect organization.

## Flake-parts batteries

### `den._.inputs'`

Exposes `inputs'` (system-qualified inputs) into aspect modules.
Source: `provides/flake-parts/inputs.nix`

### `den._.self'`

Exposes `self'` (system-qualified self outputs) into aspect modules.
Source: `provides/flake-parts/self.nix`

## Home-manager batteries

### `den._.hm-integration`

Integrates home-manager into NixOS/Darwin hosts. Imports the appropriate
home-manager module (`nixos` or `darwin`), enables
`home-manager.useGlobalPkgs` and `useUserPackages`.

### `den._.hm-os`

Merges home-manager configuration into the host OS module system. Bridges
`home-manager.users.<name>` with the user's aspect.

## Hjem batteries

### `den._.hjem-integration`

Integrates [hjem](https://github.com/feel-co/hjem) as an alternative to
home-manager. Imports hjem's NixOS module.

### `den._.hjem-os`

Merges hjem user configuration into the host. Sets `hjem.users.<name>`
from the user's aspect.

## Maid batteries

### `den._.maid-integration`

Integrates [maid](https://github.com/maid-nix/maid) as an alternative
home management system.

### `den._.maid-os`

Merges maid configuration into the host OS for each user.

## Unfree batteries

### `den._.unfree`

Allows unfree packages globally via
`nixpkgs.config.allowUnfree = true`.

### `den._.unfree-predicate`

Allows unfree packages selectively using a predicate builder. Accepts
a list of package names and builds
`nixpkgs.config.allowUnfreePredicate`.
